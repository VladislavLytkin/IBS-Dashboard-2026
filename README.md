# IBS-Dashboard-2026

Прототип школьного аналитического дашборда: **frontend (React + TS + Vite)** + **backend (Node + TS + Express)** с авторизацией, ролями, синтетическими данными за 2020–2026, отчётами в Excel и ML-прототипом прогноза риска.

> ⚠️ **Аналитика данных не затрагивается.** Папка `Data Analysis/` (ноутбуки, CSV/XLSX, документация аналитиков) к веб-части не относится и не изменяется. Веб-прототип использует собственные **синтетические** данные.

## Структура репозитория

```
Data Analysis/   # материалы аналитиков — НЕ ТРОГАЕМ (notebooks, csv/xlsx, docs)
backend/         # API: Express + TS, авторизация, роли, генерация данных, Excel-отчёты
frontend/        # SPA: React + TS + Vite, авторизация, sidebar, дашборды, отчёты
```

## Быстрый старт

Нужны два терминала.

### 1. Backend

```bash
cd backend
npm install
npm run seed     # создаёт демо-пользователей (bcrypt-хэши), настройки, уведомления, историю отчётов
npm run dev      # http://localhost:4000/api
```

> Миграции БД не требуются: для прототипа используется лёгкое JSON-хранилище
> (`backend/data/store.json`), инициализируемое командой `npm run seed`.
> Структура сущностей повторяет SQLite-схему и легко переносится на Prisma/SQLite.

### 2. Frontend

```bash
cd frontend
npm install
npm run dev      # http://localhost:5173
```

Базовый URL API задаётся в `frontend/.env` → `VITE_API_URL=http://localhost:4000/api`.

## Демо-доступы для прототипа

> Пароли в базе хранятся **только в виде bcrypt-хэшей**. Ниже — открытые пароли
> исключительно для локального прототипа; для продакшена их нужно удалить/заменить.

| Роль | Email | Пароль |
|------|-------|--------|
| ADMIN (Администратор) | `admin@school123.local` | `Admin_2026_Dashboard!` |
| DIRECTOR (Директор) | `director@school123.local` | `Director_2026_IBS!` |
| HEAD_TEACHER (Завуч) | `headteacher@school123.local` | `Zavuch_2026_School!` |
| ANALYST (Аналитик) | `analyst@school123.local` | `Analyst_2026_Data!` |
| TEACHER (Учитель) | `teacher@school123.local` | `Teacher_2026_Class!` |

На странице входа есть кнопки быстрого выбора роли.

### Права ролей

- **ADMIN** — полный доступ, включая раздел «Пользователи» (CRUD).
- **DIRECTOR** — весь дашборд, отчёты, настройки (редактирование), риски.
- **HEAD_TEACHER** — классы, ученики, риски, отчёты.
- **TEACHER** — дашборды и свои классы/ученики (без управления пользователями и настроек).
- **ANALYST** — все данные, дашборды и отчёты, без управления пользователями.

Защита маршрутов: неавторизованных перенаправляет на `/login`, при нехватке прав — на `/no-access`.

## Как проверить

1. Откройте `http://localhost:5173` → форма входа.
2. Войдите как **director** → попадёте на «Главная».
3. Слева — sidebar (сворачивается кнопкой; на мобильном открывается поверх контента).
4. Смените **Год** в шапке (2020–2026) и **Параллель** (1–11) — рейтинг/олимпиады/риски пересчитаются.
5. **Отчёты** → выберите тип/год/параллель → «Сформировать и скачать» — браузер скачает `.xlsx`.
6. Войдите как **admin** → появится пункт «Пользователи» с CRUD.
7. Войдите как **teacher** → пункт «Пользователи» скрыт, прямой переход на `/users` даёт «Нет доступа».

## Excel-отчёты

`POST /api/reports/export` формирует книгу `.xlsx` (библиотека `exceljs`) и пишет запись в историю.
Полный отчёт содержит листы: **Итоговый рейтинг**, **Риски учеников**, **Олимпиады**, **ЕГЭ-Экзамены**, **Посещаемость**.

## Синтетические данные

- Годы **2020–2026**, классы **1–11** по три литеры (А/Б/В), 20–32 ученика в классе.
- Данные **детерминированы** (стабильны между перезагрузками) и логичны: сильные классы →
  выше академический и олимпиадный балл; ниже посещаемость → выше риск и т.д.
- Предметы экзаменов зависят от параллели: ЕГЭ (10–11), ОГЭ (9), диагностические (5–8), базовые (1–4).

## ML-прогноз риска

Прототип на синтетических данных (`backend/src/utils/riskModel.ts`):
`0.35·снижение оценок + 0.30·пропуски + 0.20·низкая активность + 0.10·нет олимпиад + 0.05·отриц. динамика`,
уровни 0–33 / 34–66 / 67–100. **Не является педагогическим заключением** (предупреждение выводится в интерфейсе).

## Итоговый рейтинг

Единая формула (`backend/src/utils/scoring.ts`):
`0.35·академ + 0.25·олимпиады + 0.15·посещаемость + 0.15·активность + 0.10·(100 − риск)`,
результат 0–100, сортировка по убыванию, пересчёт по выбранному году и параллели.
