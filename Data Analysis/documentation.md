## Назначение

Датасет `synthetic_class_rating_data_v2.csv` используется для прототипа школьного аналитического дашборда. Его цель - показать, как можно рассчитывать рейтинг классов, анализировать образовательные показатели и формировать сигнал риска.

Данные являются синтетическими. Они не относятся к реальным ученикам, классам или школам.

## Единица наблюдения

Одна строка датасета соответствует одному классу за один учебный год.

Пример: строка для `7Е` описывает агрегированные показатели класса `7Е` за учебный год `2025/2026`.

## Основные поля

- `academic_year` - учебный год;
- `class_name` - название класса;
- `grade` - параллель;
- `class_letter` - буква класса;
- `students_count` - количество учеников;
- `avg_grade_5` - средняя оценка по 5-балльной шкале;
- `attendance_pct` - посещаемость в процентах;
- `homework_completion_pct` - процент выполнения домашних заданий;
- `tests_avg_pct` - средний результат контрольных и тестов;
- `behavior_score_100` - поведенческий индекс от 0 до 100;
- `olympiad_participation_pct` - доля учеников, участвующих в олимпиадах;
- `club_participation_pct` - доля учеников, участвующих в кружках и секциях;
- `parent_engagement_pct` - индекс вовлечённости родителей;
- `exam_grade_flag` - признак экзаменационной параллели;
- `rating_score_100` - итоговый рейтинг класса;
- `risk_level` - исходная категория риска.

## Ограничения

Датасет подходит для прототипирования и демонстрации логики дашборда. Его нельзя использовать для реальной оценки учеников, учителей или классов.

Для промышленного решения необходимо собрать исторические данные, определить реальное целевое событие риска, обучить модель на прошлых периодах и проверить качество на новых данных.

## Цель модели

ML-блок нужен для определения классов, которые могут требовать управленческого внимания. Модель не принимает управленческое решение сама, а формирует аналитический сигнал для дашборда.

## Постановка задачи

Задача сведена к бинарной классификации:

```text
y = 1, если risk_level = "средний" или "высокий"
y = 0, если risk_level = "низкий"
```

Следовательно, модель оценивает вероятность попадания класса в зону внимания.

## Основная модель

Основная модель - логистическая регрессия:

$$
P(y = 1 \mid X) = \frac{1}{1 + e^{-z}}
$$

$$
z = \beta_0 + \beta_1x_1 + \beta_2x_2 + \dots + \beta_nx_n
$$

Перед обучением признаки стандартизируются с помощью `StandardScaler`.

## Обоснование выбора логистической регрессии

Логистическая регрессия выбрана для первого прототипа, потому что:

1. Она возвращает вероятность, которую удобно показывать в дашборде.
2. Она интерпретируема: можно объяснить, какие признаки сильнее связаны с риском.
3. Она подходит для малого количества данных.
4. Она прозрачнее сложных моделей вроде случайного леса или нейросети.
5. Её можно использовать как понятный baseline для дальнейших улучшений.

## Метрики проверки

Для проверки качества использовались:

- `ROC-AUC`;
- `Average Precision`;
- `Balanced Accuracy`;
- `F1`;
- `Recall`;
- `Precision`.

Для задачи риска особенно важен `recall`, потому что пропуск проблемного класса хуже, чем дополнительная проверка класса с умеренным риском.

## Выход модели

Основной выход:

```text
model_attention_probability
```

Это вероятность того, что класс относится к среднему или высокому риску.

Для дашборда вероятность переводится в категории:

```text
0.00–0.30 - низкая зона внимания
0.30–0.60 - средняя зона внимания
0.60–1.00 - высокая зона внимания
```

## Ограничения

Текущая модель обучена на синтетических данных, поэтому её нельзя использовать для реальных управленческих решений.

Для реального внедрения нужно:

1. Собрать исторические данные.
2. Определить реальное целевое событие риска.
3. Обучить модель на прошлых данных.
4. Проверить качество на новых данных.
5. Откалибровать вероятности и пороги.
6. Согласовать интерпретацию результата с бизнес-заказчиком.
